# 1-80 越大越好

**分数**: 10  
**作者**: 苏汀  
**单位**: 金陵科技学院

## 题目描述
你的任务相当简单。只要用不超过 $n$ ($n \le 100$) 根火柴摆出一个尽量大的，且能被 $m$ ($m \le 3000$) 整除的正整数即可。

火柴摆放对应数字消耗：
- `1`: 2根, `2`: 5根, `3`: 5根, `4`: 4根, `5`: 5根
- `6`: 6根, `7`: 3根, `8`: 7根, `9`: 6根, `0`: 6根

### 输入格式
输入包含 $T$ 组样例。每组样例包含 2 个正整数 $n$ 和 $m$。

### 输出格式
输出共 $T$ 行，每行输出按照要求可以摆出的数的最大值；若无解则输出 -1。

## 测试命令与过程

### 测试命令
项目包含一个自动化测试脚本 `test.sh`，可以使用以下命令一键执行全部测试：
```bash

```

### 测试流程说明
1. **清理与编译**：通过 `make clean && make` 确保从最新的源代码生成 fresh 的二进制文件。
2. **自动化脚本**：`test.sh` 会将预设的 $n$ 和 $m$ 组合输入给 `matchstick` 进程，并自动对比输出结果与预期值。
3. **覆盖范围**：
    - 基础样例验证（题目给出的示例）。
    - 边界情况验证（如火柴不足、大余数等）。
    - 压力测试（$n=100, m=3000$ 的复杂场景，验证性能与正确性）。
